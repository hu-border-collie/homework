# Repository Guidelines

## 项目结构与模块组织
- `assignment/`：主要的练习答案，按题号分文件（如 `1-1.c`、`9-5.c`）。
- `exercises/`：额外练习文件（如 `P1046.c`），每个文件独立。
- 带有 `-喵.c` 后缀的文件为同题目的变体实现。
- `.vscode/`：本仓库的编辑器配置。

## 构建、测试与本地运行
- 编译单个题目：`gcc assignment/1-1.c -o 1-1`。
- 运行程序：`./1-1`（需要输入时从标准输入读取）。
- 编译练习：`gcc exercises/P1046.c -o P1046`。
- 本仓库没有统一的构建脚本或 Makefile，按文件独立编译。

## 编码风格与命名约定
- 缩进为 4 个空格，函数与控制结构的大括号另起一行。
- 每个题目保持为单一 `.c` 文件，并包含 `main()`。
- 文件命名遵循题号或章节-题号模式（如 `7-3.c`、`P1179.c`）。

## 测试指南
- 未配置自动化测试框架，主要依赖手工输入验证。
- 推荐使用输入重定向做可复现测试：`./9-5 < input.txt`。
- 记录测试输入与输出，便于复查和交流结果。

## 提交与 PR 规范
- 当前仓库缺少 Git 历史，因此没有既定提交规范。
- 新增提交建议使用简短动词开头（如 `Add 7-3 solution`），内容聚焦。
- PR 请包含改动摘要、运行方式，以及必要的输入/输出示例。

## 其他建议
- 优先使用标准 C（C11 或更新）以保证可移植性。
- 编辑器配置尽量保持最小化且与项目相关。
